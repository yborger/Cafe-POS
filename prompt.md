# POS Redesign Project Brief  
## Aurora Brew Café

---

## 1. Brand Overview

### Business Information

- **Business Name:** Aurora Brew Café  
- **Industry:** Specialty Coffee + Light Brunch  
- **Location:** Urban neighborhood with high foot traffic  

### Target Audience

- Remote workers (ages 22–40)  
- Students  
- Young professionals  
- Weekend brunch crowd  

### Brand Personality

- Warm  
- Modern  
- Calm but vibrant  
- Community-focused  
- Sustainable  

### Brand Positioning Statement

> Aurora Brew Café is a modern neighborhood coffee shop offering ethically sourced coffee, seasonal brunch, and a welcoming space for connection and productivity.

---

## 2. Visual Identity Guidelines

### Logo

- Minimal sunrise icon integrated into typography  
- Clean sans-serif logotype  
- Flat design, scalable  

### Color Palette

#### Primary Colors

- **Deep Espresso Brown** — `#3B2F2F`  
- **Soft Sunrise Coral** — `#FF6B4A`  
- **Cream Oat Milk** — `#F4EFEA`  

#### Secondary Colors

- **Sage Green** — `#8FAF9A`  
- **Charcoal Gray** — `#2B2B2B`  
- **Soft Sky** — `#A8DADC`  

### Typography

- **Headings:** Modern geometric sans-serif (e.g., Poppins / Avenir style)  
- **Body:** Clean sans-serif (Inter / Helvetica style)  
- **Buttons:** Bold, uppercase  

### Tone of Voice

- Friendly but not childish  
- Calm and reassuring  
- Minimal emoji use  
- Clear, concise language  

---

## 3. Current POS Problems

### System Issues

The current POS system:

- Looks outdated  
- Has too many nested menus  
- Slows down during rush hour  
- Has confusing drink modifiers  
- Poor visual hierarchy  
- Difficult for new staff to learn  
- Does not visually match the brand  

### Key Pain Points

- Baristas struggle during peak hours  
- Order errors due to modifier confusion  
- Checkout flow requires too many taps  
- No dark mode option  
- Poor readability in bright café lighting  

---

## 4. POS Redesign Goals

### Primary Goals

- Speed up ordering process  
- Reduce training time for new employees  
- Improve visual clarity  
- Align with Aurora Brew Café branding  
- Create an intuitive modifier system  
- Support touch and tablet interfaces  

### Secondary Goals

- Enable scalability for future franchise growth  
- Include analytics dashboard  
- Improve accessibility (WCAG compliance)  
- Add dark mode  

---

## 5. Core Features Required

### 5.1 Order Interface

- Large category buttons:
  - Coffee  
  - Tea  
  - Pastries  
  - Breakfast  
  - Lunch  
  - Retail  
- Grid layout (product photos optional)  
- Quick-add favorites  
- Drag-to-cart functionality  
- Visible cart on right side  

---

### 5.2 Drink Customization

- Milk options: Oat, Almond, Soy, Whole  
- Temperature: Hot / Iced  
- Size: S / M / L  
- Sweetness level slider  
- Extra shots toggle  
- Visual confirmation before adding to cart  

---

### 5.3 Checkout

**Payment Types:**

- Card  
- Contactless  
- Mobile Pay  
- Cash  
- Split payment option  

**Additional Features:**

- Tip selection screen  
- Receipt options:
  - Print  
  - Email  
  - SMS  

---

### 5.4 Admin Dashboard

- Inventory tracking  
- Sales reports  
- Peak hour analytics  
- Employee performance tracking  
- Menu editor  
- Price adjustment tool  

---

### 5.5 Hardware Compatibility

- iPad & touchscreen terminals  
- Receipt printer  
- Barcode scanner  
- Cash drawer  
- Kitchen display system integration  

---

## 6. UX Requirements

- Maximum **3 taps** to complete a standard drink order  
- Clear visual hierarchy  
- High contrast for readability  
- Responsive design  
- Large touch targets (minimum 44px)  
- Loading feedback animations  
- Offline mode capability  
- Role-based access:
  - Barista  
  - Manager  
  - Admin  

---

## 7. UI Style Direction

- Clean, airy layout  
- Rounded buttons (8px–12px radius)  
- Soft shadows (subtle depth)  
- Minimal gradients  
- Card-based design system  
- Consistent iconography  
- Microinteractions for feedback  

### Dark Mode

- **Background:** Charcoal  
- **Accent:** Sunrise Coral  
- **Text:** Cream  

---

## 8. Accessibility Requirements

- WCAG AA compliant contrast  
- Color-blind friendly palette  
- Adjustable text size  
- Screen reader compatibility  
- Clear icon labeling  

---

## 9. Deliverables

- Accessibility documentation
